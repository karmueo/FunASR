# FunASR 服务使用方法

本文档说明如何在仓库根目录通过 `python -m funasr_server.app` 启动本项目的
FastAPI + Gradio 转写服务，并调用 HTTP 接口完成音频转写。

## 一、运行前准备

### 1. 进入仓库根目录

```bash
cd /home/tl/work/FunASR
```

### 2. 安装 Python 依赖

建议先使用虚拟环境，避免污染系统 Python 环境。

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -r funasr_server/requirements.txt
```

风险提示：

- 安装依赖可能触发网络下载，并占用本地磁盘空间。
- 如果依赖安装后想回滚，退出虚拟环境并删除 `.venv` 目录即可：

```bash
deactivate
rm -rf .venv
```

### 3. 准备模型与 GPU 环境

默认配置会使用：

- ASR/VAD/标点/说话人模型运行在 `cuda:0`
- 翻译模型运行在 `cuda:1`
- 翻译模型路径为 `/home/tl/models/hunyuan-mt-7b`

如果你的机器没有第二张 GPU，或者本地没有该翻译模型目录，建议在启动前通过环境变量调整配置，
否则服务初始化可能失败。

## 二、启动服务

### 1. 直接使用默认配置启动

在仓库根目录执行：

```bash
python -m funasr_server.app
```

启动成功后，终端会输出类似地址：

- FastAPI 服务地址：`http://0.0.0.0:8000`
- API 文档地址：`http://0.0.0.0:8000/docs`
- Gradio 页面地址：`http://0.0.0.0:8001`

如果是在本机浏览器访问，通常使用：

- `http://127.0.0.1:8000/docs`
- `http://127.0.0.1:8001`

### 2. 使用环境变量覆盖配置后启动

所有配置字段都支持用 `FUNASR_` 前缀环境变量覆盖，例如：

```bash
export FUNASR_HOST=0.0.0.0
export FUNASR_PORT=8000
export FUNASR_DEVICE=cuda:0
export FUNASR_DEVICE_TRANSLATION=cuda:1
export FUNASR_TRANSLATION_MODEL=/home/tl/models/hunyuan-mt-7b
export FUNASR_ENABLE_TRANSLATION=true

python -m funasr_server.app
```

单 GPU 或暂时不需要翻译功能时，可以这样启动：

```bash
export FUNASR_DEVICE=cuda:0
export FUNASR_ENABLE_TRANSLATION=false

python -m funasr_server.app
```

只用 CPU 调试时，可以这样启动：

```bash
export FUNASR_DEVICE=cpu
export FUNASR_ENABLE_TRANSLATION=false

python -m funasr_server.app
```

## 三、常用环境变量说明

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `FUNASR_HOST` | `0.0.0.0` | FastAPI 和 Gradio 监听地址 |
| `FUNASR_PORT` | `8000` | FastAPI 端口，Gradio 端口会自动使用 `PORT+1` |
| `FUNASR_DEVICE` | `cuda:0` | ASR / VAD / 标点 / 说话人模型使用的设备 |
| `FUNASR_DEVICE_TRANSLATION` | `cuda:1` | 翻译模型使用的设备 |
| `FUNASR_ENABLE_TRANSLATION` | `true` | 是否启用自动翻译为中文 |
| `FUNASR_TRANSLATION_MODEL` | `/home/tl/models/hunyuan-mt-7b` | 翻译模型路径或模型名 |
| `FUNASR_ASR_MODEL` | `iic/SenseVoiceSmall` | ASR 模型名 |
| `FUNASR_VAD_MODEL` | `fsmn-vad` | VAD 模型名 |
| `FUNASR_PUNC_MODEL` | `ct-punc` | 标点模型名 |
| `FUNASR_SPK_MODEL` | `cam++` | 说话人模型名 |
| `FUNASR_HUB` | `ms` | 模型来源 |
| `FUNASR_MAX_SINGLE_SEGMENT_TIME` | `60000` | VAD 单段最大时长，单位毫秒 |
| `FUNASR_BATCH_SIZE_S` | `300` | 动态批大小，单位秒 |
| `FUNASR_BATCH_SIZE_THRESHOLD_S` | `60` | 单段最大时长阈值，单位秒 |
| `FUNASR_MERGE_THR` | `0.5` | 说话人聚类合并阈值 |
| `FUNASR_TRANSLATION_MAX_LENGTH` | `512` | 翻译生成最大长度 |
| `FUNASR_MAX_AUDIO_DURATION_S` | `7200` | 允许处理的最大音频时长，单位秒 |
| `FUNASR_REQUEST_TIMEOUT_S` | `600` | HTTP keep-alive 超时时间，单位秒 |

## 四、接口调用方法

### 1. 健康检查接口

```bash
curl http://127.0.0.1:8000/health
```

正常情况下会返回类似结果：

```json
{
  "status": "ok",
  "models_loaded": true
}
```

### 2. 音频转写接口

接口地址：`POST /transcribe`

表单字段说明：

| 字段名 | 必填 | 说明 |
| --- | --- | --- |
| `audio` | 是 | 待转写音频文件，支持 `.wav`、`.mp3`、`.flac`、`.ogg`、`.m4a` |
| `language` | 否 | 语言代码，默认 `auto` 自动检测 |
| `speaker_num` | 否 | 说话人数，留空表示由模型自动判断 |

示例命令：

```bash
curl -X POST http://127.0.0.1:8000/transcribe \
  -F "audio=@/path/to/test.wav" \
  -F "language=auto"
```

如果已知说话人数，例如 2 个人：

```bash
curl -X POST http://127.0.0.1:8000/transcribe \
  -F "audio=@/path/to/test.wav" \
  -F "language=auto" \
  -F "speaker_num=2"
```

返回结果中通常包含：

- `task_id`：本次请求 ID
- `language`：识别到的语言
- `duration_s`：音频时长估计值
- `speakers`：说话人列表，例如 `["spk0", "spk1"]`
- `segments`：分句结果，每段包含开始/结束时间、说话人、原文、语言、中文翻译 `text_zh`

## 五、Gradio 页面使用方法

启动服务后，直接打开：

```text
http://127.0.0.1:8001
```

在页面中上传音频文件后即可发起转写。Gradio 页面和 HTTP API 共用同一个已加载的
Pipeline，所以首次启动时会先加载模型，后续请求会复用内存中的模型实例。

## 六、停止服务

在启动服务的终端窗口按 `Ctrl+C` 即可停止服务。

当前启动入口已经注册了 `SIGINT` / `SIGTERM` 信号处理，会尽量先关闭 Gradio，
再通知 Uvicorn 退出。

## 七、常见问题与处理方式

### 1. 启动时报翻译模型路径不存在

如果机器上没有 `/home/tl/models/hunyuan-mt-7b`，可以改成你自己的模型路径：

```bash
export FUNASR_TRANSLATION_MODEL=/your/model/path
python -m funasr_server.app
```

如果暂时不需要翻译功能，直接关闭翻译：

```bash
export FUNASR_ENABLE_TRANSLATION=false
python -m funasr_server.app
```

回滚方式：

```bash
unset FUNASR_TRANSLATION_MODEL
unset FUNASR_ENABLE_TRANSLATION
```

### 2. 机器只有一张 GPU，但默认翻译设备是 `cuda:1`

可以把翻译模型也放到 `cuda:0`，或者直接关闭翻译：

```bash
export FUNASR_DEVICE=cuda:0
export FUNASR_DEVICE_TRANSLATION=cuda:0
python -m funasr_server.app
```

回滚方式：

```bash
unset FUNASR_DEVICE
unset FUNASR_DEVICE_TRANSLATION
```

### 3. 端口 8000 或 8001 已被占用

修改 `FUNASR_PORT` 即可，Gradio 端口会自动变成 `FUNASR_PORT + 1`：

```bash
export FUNASR_PORT=18000
python -m funasr_server.app
```

此时访问地址变为：

- API 文档：`http://127.0.0.1:18000/docs`
- Gradio 页面：`http://127.0.0.1:18001`

回滚方式：

```bash
unset FUNASR_PORT
```

### 4. 远程机器上启动后，本机浏览器打不开页面

请优先确认以下几项：

- 服务端是否已监听 `0.0.0.0`
- 云服务器/防火墙是否放行了 API 端口和 Gradio 端口
- 浏览器访问时是否使用了服务器真实 IP，而不是 `127.0.0.1`

风险提示：

- 放开公网端口会带来暴露服务的风险。
- 如果只是临时测试，建议测试完成后及时收回安全组/防火墙放行规则。
